While building accessible online services, the following guidelines were followed on projects

1) Web Content Accessibility Guidelines WCAG2.1

Please click the below links for work examples adhering to WCAG2.1 principles of

  • Perceivable

2) Semantic coding for building accessible services

  • Referred to the following websites
    Accessibility through semantic html
    Semantic Markup
  • Hypertext Markup Language HTML for displaying content
  • Cascading style sheets for look and feel of the site
  • The hypothesis is such that keeping HTML tags for displaying content makes it easier for assistive technologies to pick up the relevant html tags to announce it to the end users who use assistive technologies. I haven't tested this with any assitive technologies till date but the hypothesis is based on discussions during team meetings on projects and reading through articles in free time.
  • The assistive technologies considered in scope were that of Dragon Speaking Software, Screen Readers, Screen Magnifiers, and Speech browsers
